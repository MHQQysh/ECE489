## 观察要点

### 时间轴

```
0s ────────────────> 5s ──> 5.1s ────────────────> 15s
   正常行走           ⚡推力   恢复平衡              结束
```

### 1. 前 5 秒：正常行走

- 机器人以 0.5 m/s 速度前进
- 观察步态是否稳定
- 身体高度保持在 ~0.32m

### 2. 5.0-5.1 秒：施加推力

**你会看到：**
- ⚡ 机器人突然被推向一侧（Y 方向）
- 身体会倾斜
- 可能会有一条腿离地
- 控制台会显示：`⚡⚡⚡ 在 5.00s 施加 40N 侧向推力！`

### 3. 5.1 秒后：恢复过程

**观察机器人如何恢复：**
- 调整腿部位置
- 重新找回平衡
- 身体逐渐回到正常高度
- 继续前进

**成功恢复的标志：**
- ✅ 身体高度恢复到 0.30m 以上
- ✅ 身体不再剧烈晃动
- ✅ 继续向前行走

**失败的标志：**
- ❌ 身体高度低于 0.15m
- ❌ 机器人倒地
- ❌ 无法继续行走

## 调整推力参数

如果你想测试不同强度的推力，可以修改脚本中的参数：

```python
# 在 demo_push_test.py 中修改这些参数

# 推力大小（N）
push_force = 40.0  # 改为 30.0 (轻), 50.0 (中), 60.0 (强)

# 推力方向
data.xfrc_applied[trunk_body_id, 1] = push_force  # Y 方向（侧向）
# 改为：
# data.xfrc_applied[trunk_body_id, 0] = push_force  # X 方向（前后）
# data.xfrc_applied[trunk_body_id, 2] = push_force  # Z 方向（上下）

# 推力时间
push_time = 5.0  # 改为 3.0, 7.0 等

# 推力持续时间
push_duration = 0.1  # 改为 0.05 (短), 0.2 (长)
```

## 推力强度参考

| 推力 (N) | 相当于 | 预期效果 |
|---------|--------|---------|
| 20 N | 2 kg 重量 | 轻微晃动，容易恢复 |
| 30 N | 3 kg 重量 | 明显倾斜，需要调整 |
| **40 N** | **4 kg 重量** | **显著推动，考验控制器** (默认) |
| 50 N | 5 kg 重量 | 强力推动，可能摔倒 |
| 60 N | 6 kg 重量 | 极强推力，很难恢复 |

**参考**：Go2 机器人重量 = 6 kg

## 在 MuJoCo Viewer 中的操作

### 相机控制
- **鼠标左键拖动**: 旋转视角
- **鼠标右键拖动**: 平移视角
- **鼠标滚轮**: 缩放
- **双击**: 重置视角

### 暂停/继续
- **空格键**: 暂停/继续仿真

### 慢动作
- **[**: 减慢仿真速度
- **]**: 加快仿真速度

### 建议视角
1. **侧视图**: 从侧面观察推力效果
2. **俯视图**: 从上方观察横向位移
3. **跟随视角**: 跟随机器人移动

## 故障排除

### 问题 1: 窗口无法打开
```bash
# 检查是否有显示
echo $DISPLAY

# 如果是远程连接，需要 X11 转发
ssh -X user@host
```

### 问题 2: 机器人立即摔倒
- 可能是初始状态不稳定
- 尝试减小推力（30N）
- 检查关节范围设置

### 问题 3: 推力效果不明显
- 增大推力（50N, 60N）
- 延长推力持续时间（0.2s）
- 检查推力方向是否正确
