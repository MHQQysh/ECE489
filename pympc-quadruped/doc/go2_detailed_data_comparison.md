## 8. 关键差异总结

### 8.1 ✅ 完全相同的部分
- 所有刚体的惯性参数（质量、质心、惯性矩）
- 所有刚体的位置和姿态
- 基座和腿部的碰撞几何体尺寸和位置（除足部微小偏移）
- 视觉网格文件（16个 OBJ 文件）

### 8.2 ⚠️ 主要差异

| 项目 | robot/go2 | unitree_go2 |
|------|-----------|-------------|
| **Hip 关节范围** | ±70° | ±60° |
| **Thigh 关节范围** | 无限制 | 前腿: -90°~200°<br>后腿: -30°~260° |
| **碰撞 condim** | 3 (点接触) | 1 (默认), 6 (足部) |
| **摩擦系数** | 0.8, 0.3, 0.3 | 0.4 (默认), 0.4/0.02/0.01 (足部) |
| **执行器扭矩限制** | 无限制 | Hip/Thigh: ±23.7 Nm<br>Calf: ±45.43 Nm |
| **关节动力学** | 未定义 | damping=0.1, armature=0.01, frictionloss=0.2 |
| **触觉传感器** | ✅ 4个 | ❌ 无 |
| **力矩传感器** | ❌ 无 | ✅ 12个 (带噪声) |
| **关键帧** | ❌ 无 | ✅ "home" 姿态 |
| **Python 配置** | ❌ 无 | ✅ 完整的配置管理 |

### 8.3 功能互补

**robot/go2/go2.xml 独有:**
- 触觉传感器（足部接触检测）
- 完整的场景元素（地面、光照、相机）
- URDF 格式支持

**unitree_go2/xmls/scene_go2.xml 独有:**
- 关节力矩传感器（带噪声模拟）
- 执行器扭矩限制
- 关节动力学参数
- 关键帧定义
- 更精细的碰撞配置

**unitree_go2/go2_constants.py 独有:**
- 程序化配置管理
- PD 控制器参数
- 灵活的碰撞配置切换
- 与 mjlab 框架集成

## 9. 使用建议

### 9.1 选择 robot/go2/go2.xml 当需要：
- 触觉传感器数据（足部接触力）
- 更大的关节活动范围（特别是 hip 关节）
- 独立的、完整的仿真场景
- URDF 格式（ROS 集成）

### 9.2 选择 unitree_go2/xmls/scene_go2.xml 当需要：
- 关节力矩传感器（执行器反馈）
- 执行器扭矩限制（更真实的硬件约束）
- 关节动力学参数（阻尼、摩擦损失）
- 关键帧（快速重置到标准姿态）
- 更精细的碰撞模型（condim=6）

### 9.3 选择 unitree_go2/go2_constants.py 当需要：
- Python 项目集成
- 程序化配置执行器参数
- 灵活切换碰撞配置
- 与 mjlab 框架配合使用
