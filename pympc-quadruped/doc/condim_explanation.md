# condim (接触维度) 详细解释

## 1. 什么是 condim？

**condim = Contact Dimension（接触维度）**

它定义了 MuJoCo 物理引擎在计算两个物体接触时，考虑多少个自由度的力和力矩。

简单来说：**condim 决定了接触点能产生多少种不同方向的力。**

---

## 2. condim 的不同值

### condim = 1（仅法向力）

**含义：** 只计算垂直于接触面的力（法向力）

**自由度：** 1 个
- Fz（垂直接触面的力）

**物理意义：**
- 两个物体只能"推"，不能"拉"
- 没有摩擦力
- 最简单、最快的接触模型

**适用场景：**
- 非关键的碰撞（如机器人身体与墙壁）
- 不需要摩擦的接触
- 优化性能时使用

**示例：**
```
地面
  ↑ Fz (法向力)
  │
  └─ 物体
  
只有向上的支撑力，物体会在地面上自由滑动（无摩擦）
```

---

### condim = 3（点接触 + 2D 摩擦）

**含义：** 法向力 + 两个方向的摩擦力

**自由度：** 3 个
- Fz（法向力）
- Fx（切向摩擦力 1）
- Fy（切向摩擦力 2）

**物理意义：**
- 标准的点接触模型
- 有摩擦，物体不会随意滑动
- 最常用的接触模型

**适用场景：**
- 足部与地面接触
- 大多数机器人接触
- 平衡性能和精度

**示例：**
```
地面
  ↑ Fz (法向力)
  │
  ├→ Fx (前后摩擦)
  │
  ├→ Fy (左右摩擦)
  │
  └─ 物体
  
物体可以站稳，不会滑动
```

---

### condim = 4（condim=3 + 滚动摩擦）

**含义：** condim=3 + 滚动阻力

**自由度：** 4 个
- Fz（法向力）
- Fx, Fy（切向摩擦）
- Tr（滚动阻力矩）

**物理意义：**
- 考虑物体滚动时的阻力
- 适合球形或圆柱形物体

**适用场景：**
- 轮子
- 球形足部
- 需要考虑滚动的场景

---

### condim = 6（完整接触）

**含义：** 完整的 3D 力 + 3D 力矩

**自由度：** 6 个
- Fx, Fy, Fz（3D 力）
- Tx, Ty, Tz（3D 力矩）

**物理意义：**
- 最精确的接触模型
- 考虑接触面积和旋转阻力
- 可以模拟"粘"在地面上的感觉

**适用场景：**
- 需要极高精度的足部接触
- 面接触（不是点接触）
- 需要考虑旋转阻力的场景

**示例：**
```
地面
  ↑ Fz (法向力)
  │
  ├→ Fx, Fy (切向摩擦)
  │
  ├↻ Tx, Ty (倾斜力矩)
  │
  ├↻ Tz (旋转力矩)
  │
  └─ 物体
  
物体不仅不会滑动，还会抵抗旋转
```

## 3. 直观对比

### 想象一个机器人足部踩在地面上

**condim = 1:**
```
足部像在冰面上
- 可以站立（有支撑力）
- 但会到处滑动（无摩擦）
- 可以自由旋转
```

**condim = 3:**
```
足部像在普通地面上
- 可以站立
- 不会滑动（有摩擦）
- 但可以像陀螺一样原地旋转
```

**condim = 6:**
```
足部像粘在地面上
- 可以站立
- 不会滑动
- 也不容易旋转（有旋转阻力）
- 最真实的接触
```

## 4. 性能对比

| condim | 计算量 | 精度 | 稳定性 | 适用场景 |
|--------|--------|------|--------|---------|
| **1** | 最快 | 最低 | 高 | 非关键碰撞 |
| **3** | 快 | 中等 | 高 | 标准接触 |
| **4** | 中等 | 中高 | 中 | 滚动物体 |
| **6** | 最慢 | 最高 | 中 | 精确足部接触 |

## 5. Go2 的 condim 配置

### 修改前（robot/go2/go2.xml 原始）

```xml
<default class="collision">
  <geom priority="1" condim="3" friction="0.8 0.3 0.3" group="3"/>
</default>
```

**所有碰撞几何体都是 condim=3**
- 优点：简单统一
- 缺点：非关键碰撞也用 condim=3，浪费计算

### 修改后（与 unitree_go2 一致）

```xml
<default class="go2">
  <geom friction="0.4" margin="0.001" condim="1"/>  <!-- 默认 -->
  <default class="collision">
    <geom group="3"/>
    <default class="foot">
      <geom priority="1" condim="6" friction="0.4 0.02 0.01"/>  <!-- 足部 -->
    </default>
  </default>
</default>
```

**分层配置：**
- 默认几何体：condim=1（快速）
- 足部：condim=6（精确）

---

## 6. 为什么 Go2 使用分层 condim？

### 机器人身体碰撞（condim=1）

```
场景：机器人身体碰到墙壁
需求：只需要知道"碰到了"，推开即可
使用：condim=1（仅法向力）
好处：计算快，够用
```

### 足部接触（condim=6）

```
场景：足部踩在地面上
需求：
  - 不能滑动（需要摩擦）
  - 不能随意旋转（需要旋转阻力）
  - 精确的接触力（用于控制）
使用：condim=6（完整接触）
好处：最精确，最真实
```

## 7. condim=6 的实际效果

### 测试代码

```python
import mujoco
import numpy as np

model = mujoco.MjModel.from_xml_path('robot/go2/go2.xml')
data = mujoco.MjData(model)

# 仿真一步
mujoco.mj_step(model, data)

# 查看接触信息
for i in range(data.ncon):
    contact = data.contact[i]
    
    # condim=6 提供 6 个分量
    force = contact.force[:6]
    
    print(f"接触 {i}:")
    print(f"  法向力: {force[0]:.2f} N")
    print(f"  切向力 X: {force[1]:.2f} N")
    print(f"  切向力 Y: {force[2]:.2f} N")
    print(f"  力矩 X: {force[3]:.4f} Nm")
    print(f"  力矩 Y: {force[4]:.4f} Nm")
    print(f"  力矩 Z: {force[5]:.4f} Nm")
```

### 输出示例

```
接触 0 (FL 足部):
  法向力: 17.23 N        ← 支撑机器人重量
  切向力 X: 0.15 N       ← 前后摩擦力
  切向力 Y: -0.08 N      ← 左右摩擦力
  力矩 X: 0.0012 Nm      ← 绕 X 轴的阻力
  力矩 Y: -0.0008 Nm     ← 绕 Y 轴的阻力
  力矩 Z: 0.0003 Nm      ← 绕 Z 轴的旋转阻力
```

**有了这些信息，控制器可以：**
- 知道每只脚承受多少重量
- 检测是否在滑动
- 检测足部是否在旋转
- 更精确地控制平衡

## 8. condim 与摩擦系数的关系

### condim=1 时

```xml
<geom condim="1" friction="0.4 0.02 0.01"/>
```

**摩擦系数被忽略！** 因为 condim=1 不计算摩擦力。

### condim=3 时

```xml
<geom condim="3" friction="0.4 0.02 0.01"/>
```

**只使用 mu1=0.4**（滑动摩擦），mu2 和 mu3 被忽略。

### condim=6 时

```xml
<geom condim="6" friction="0.4 0.02 0.01"/>
```

**使用所有三个摩擦系数：**
- mu1=0.4: 滑动摩擦
- mu2=0.02: 滚动摩擦
- mu3=0.01: 扭转摩擦

## 9. 常见问题

### Q1: 为什么不全部用 condim=6？

**A:** 
- condim=6 计算量大
- 非关键碰撞不需要这么精确
- 分层配置平衡性能和精度

### Q2: condim=1 会让机器人滑倒吗？

**A:**
- 如果足部是 condim=1，会滑倒
- 但 Go2 的足部是 condim=6，不会滑倒
- 只有非关键部位（身体）是 condim=1

### Q3: condim=3 和 condim=6 哪个更好？

**A:**
- condim=3: 更快，适合大多数情况
- condim=6: 更精确，适合需要高精度的场景
- Go2 足部用 condim=6 是为了最高精度

### Q4: 如何选择 condim？

**A:**
```
非关键碰撞 → condim=1 (快速)
标准接触   → condim=3 (平衡)
精确接触   → condim=6 (精确)
滚动物体   → condim=4 (特殊)
```

## 10. 总结

### condim 简单记忆

```
condim = 1: 只能"推"，会滑动
condim = 3: 能"推"能"抓"，不滑动
condim = 6: 能"推"能"抓"能"粘"，最真实
```

### Go2 的配置策略

```
身体碰撞: condim=1 (够用就好，省计算)
足部接触: condim=6 (要最精确，不省计算)
```

### 为什么这样配置？

**因为足部接触是最重要的：**
- 决定机器人是否稳定
- 决定能否正确行走
- 需要最精确的接触信息

**而身体碰撞不重要：**
- 只是偶尔碰到障碍物
- 只需要知道"碰到了"
- 不需要精确的力信息
