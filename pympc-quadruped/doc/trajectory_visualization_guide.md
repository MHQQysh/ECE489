## 👀 观察要点

### 1. 正常行走阶段 (0-5秒)

**身体轨迹（红色）**：
- 应该是一条相对平直的线
- 沿 X 轴方向前进
- 高度保持稳定

**足端轨迹（彩色）**：
- 呈现周期性的弧线
- 摆动相：足端抬起，形成弧线
- 支撑相：足端接触地面，轨迹平直
- 对角腿同步（FL+RR, FR+RL）

### 2. 推力施加阶段 (5.0-5.1秒)

**你会看到**：
- ⚡ 红色大球出现在机器人身体上
- 🟠 橙色箭头指向侧方（Y 方向）
- 身体轨迹突然偏离
- 足端轨迹变得不规则

**预期效果**：
- 机器人向侧面倾斜
- 身体轨迹出现横向位移
- 某些足端可能离地

### 3. 恢复阶段 (5.1-15秒)

**观察恢复过程**：
- 身体轨迹逐渐回到直线
- 足端轨迹恢复规律性
- 步态重新稳定

**成功恢复的标志**：
- ✅ 身体轨迹重新变直
- ✅ 足端轨迹恢复周期性
- ✅ 机器人继续前进

## 🔬 深入分析

### 观察步态模式

**对角步态（Trotting）**：
- FL + RR 同时摆动
- FR + RL 同时摆动
- 两组交替

**在轨迹中的表现**：
- 绿色(FL) 和 紫色(RR) 轨迹同步
- 蓝色(FR) 和 黄色(RL) 轨迹同步

### 观察推力影响

**身体轨迹变化**：
```
正常:  ────────────────────>
推力:  ──────────╱
恢复:  ──────────╱─────────>
```

**足端轨迹变化**：
- 支撑腿：承受更大压力，轨迹下沉
- 摆动腿：可能提前着地或延迟抬起

## 🎯 实验建议

### 实验 1: 不同推力大小

修改 `push_force` 参数：
```python
push_force = 30.0  # 轻推
push_force = 40.0  # 中等（默认）
push_force = 50.0  # 强推
push_force = 60.0  # 猛推
```

**观察**：推力越大，轨迹偏离越明显

### 实验 2: 不同推力方向

修改推力方向：
```python
# 侧向（默认）
data.xfrc_applied[trunk_body_id, 1] = push_force

# 前向
data.xfrc_applied[trunk_body_id, 0] = push_force

# 后向
data.xfrc_applied[trunk_body_id, 0] = -push_force
```

### 实验 3: 不同速度

修改期望速度：
```python
vel_base_des = np.array([0.3, 0.0, 0.0])  # 慢速
vel_base_des = np.array([0.5, 0.0, 0.0])  # 中速（默认）
vel_base_des = np.array([0.8, 0.0, 0.0])  # 快速
```

**观察**：速度越快，恢复越困难

## 🐛 故障排除

### 问题 1: 看不到轨迹

**原因**：轨迹太细或颜色太淡

**解决**：
- 调整相机距离（滚轮缩放）
- 修改轨迹大小（增大 `size` 参数）

### 问题 2: 轨迹太密集

**原因**：记录频率太高

**解决**：
```python
# 修改记录频率
if step % 10 == 0:  # 改为 10, 20, 30
```

### 问题 3: 性能问题

**原因**：轨迹点太多

**解决**：
- 减少记录频率
- 缩短演示时间
- 限制轨迹点数量
