# Go2 关节限制修复

## 问题描述

Go2 的 MJCF 文件缺少 **thigh 关节的角度限制**，这会导致：
1. 关节可以无限制旋转，超出物理可能的范围
2. 仿真中可能出现不合理的姿态
3. 控制器无法正确处理关节限制

## 修复内容

### 修复前
```xml
<!-- 只有 hip 和 calf 有限制 -->
<joint name="FL_hip_joint" axis="1 0 0" range="-1.22173 1.22173"/>
<joint name="FL_thigh_joint" axis="0 1 0"/>  <!-- 无限制！ -->
<joint name="FL_calf_joint" axis="0 1 0" range="-2.7227 -0.83776"/>
```

### 修复后
```xml
<!-- 所有关节都有正确的限制 -->
<joint name="FL_hip_joint" axis="1 0 0" range="-1.0472 1.0472"/>
<joint name="FL_thigh_joint" axis="0 1 0" range="-1.5708 3.4907"/>
<joint name="FL_calf_joint" axis="0 1 0" range="-2.7227 -0.83776"/>
```

## 完整的关节限制表

### Hip 关节（所有腿相同）
- **范围**: -1.0472 到 1.0472 rad
- **角度**: -60° 到 +60°
- **说明**: 髋关节外展/内收

### Front Thigh 关节（FL, FR）
- **范围**: -1.5708 到 3.4907 rad
- **角度**: -90° 到 +200°
- **说明**: 前腿大腿关节，范围较大

### Rear Thigh 关节（RL, RR）
- **范围**: -0.5236 到 4.5379 rad
- **角度**: -30° 到 +260°
- **说明**: 后腿大腿关节，范围更大

### Calf 关节（所有腿相同）
- **范围**: -2.7227 到 -0.83776 rad
- **角度**: -156° 到 -48°
- **说明**: 小腿关节，始终为负（膝盖向后弯）

## 关键发现

### 1. Hip 关节限制更新
- **修复前**: ±1.22173 rad (±70°)
- **修复后**: ±1.0472 rad (±60°)
- **原因**: 根据 unitree_go2 官方配置，Go2 的髋关节限制更严格

### 2. 前后腿不对称
Go2 的前腿和后腿有不同的 thigh 关节限制：
- **前腿**: -90° 到 +200° (范围 290°)
- **后腿**: -30° 到 +260° (范围 290°)

这种不对称设计允许：
- 前腿向前伸展更多
- 后腿向后伸展更多
- 更灵活的步态和姿态

### 3. 与 Aliengo 的对比

| 关节 | Aliengo | Go2 |
|------|---------|-----|
| Hip | ±70° | ±60° |
| Thigh | 无限制 | 前: -90°~+200°<br>后: -30°~+260° |
| Calf | -159°~-37° | -156°~-48° |

**关键差异**:
- Aliengo 的 thigh 关节无限制（可能是简化模型）
- Go2 有明确的、不对称的 thigh 关节限制
- Go2 的 calf 关节可以弯曲更多（-48° vs -37°）

3. 不会出现不自然的姿态

## 为什么这很重要

### 1. 物理真实性
关节限制反映了真实机器人的物理约束：
- 机械结构的限制
- 避免自碰撞
- 保护电机和传动系统

### 2. 控制稳定性
正确的关节限制帮助控制器：
- 避免规划不可达的轨迹
- 在接近限制时调整行为
- 防止关节"卡死"

### 3. 仿真准确性
有限制的关节使仿真更接近真实机器人：
- 相同的工作空间
- 相同的运动学约束
- 更准确的动力学行为

确保 `reset_robot_state` 中的初始关节角度在限制范围内：
```python
# 检查 utils/mujoco_simulation_utils.py
q_pos_init = np.array([
    0, 0, robot_config.base_height_des,
    1, 0, 0, 0,
    0.1, 0.8, -1.5,   # FL: 检查 0.8 是否在 [-1.5708, 3.4907] 内 ✓
    -0.1, 0.8, -1.5,  # FR: 检查 0.8 是否在 [-1.5708, 3.4907] 内 ✓
    0.1, 1.0, -1.5,   # RL: 检查 1.0 是否在 [-0.5236, 4.5379] 内 ✓
    -0.1, 1.0, -1.5   # RR: 检查 1.0 是否在 [-0.5236, 4.5379] 内 ✓
])
```
✅ 所有初始角度都在限制范围内

### 2. 监控关节状态
在仿真中监控关节是否接近限制：
```python
# 在控制循环中添加
for i, (q, limit) in enumerate(zip(joint_positions, joint_limits)):
    if abs(q - limit[0]) < 0.1 or abs(q - limit[1]) < 0.1:
        print(f"Warning: Joint {i} near limit: {q} (limit: {limit})")
```

### 3. 调整控制器
如果关节经常接近限制，考虑：
- 调整步态参数
- 降低期望速度
- 修改足部落点规划
