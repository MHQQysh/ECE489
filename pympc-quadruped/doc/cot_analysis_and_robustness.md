# CoT 值分析和鲁棒性测试说明

## 1. CoT 值分析 - 为什么这么高？

### 我们的结果 vs 文献值

| 对象 | CoT 值 | 说明 |
|------|--------|------|
| **人类行走** | 0.2 | 最高效 |
| **人类跑步** | 0.8 | |
| **四足动物 (马、狗)** | 0.5-0.8 | 自然进化优化 |
| **MIT Cheetah** | 0.5-0.8 | 高性能机器人 |
| **ANYmal** | 0.8-1.2 | 商业四足机器人 |
| **Spot (Boston Dynamics)** | 1.0-1.5 | 商业四足机器人 |
| **我们的 Go2 MPC** | **4.9-6.7** | ❌ **偏高 5-10 倍！** |

---

### 问题根源：行走距离太短

#### 实际数据分析

```
实验: 平地 0.5 m/s, 10 秒

期望行走距离: 0.5 m/s × 10s = 5.0 m
实际行走距离: 2.28 m
完成率: 45.6% ❌

总能量消耗: 963.82 J
平均功率: 96.38 W

CoT 计算:
  CoT = 963.82 / (6.004 kg × 9.81 m/s² × 2.28 m)
  CoT = 963.82 / 134.18
  CoT = 6.23
```

#### 如果行走距离正常

```
假设: 实际走了 5.0 m (期望距离)

CoT = 963.82 / (6.004 × 9.81 × 5.0)
CoT = 963.82 / 294.30
CoT = 3.27  ✅ 更接近合理值

假设: 实际走了 5.0 m 且能量消耗减半

CoT = 481.91 / (6.004 × 9.81 × 5.0)
CoT = 481.91 / 294.30
CoT = 1.64  ✅ 接近商业机器人水平
```

---

### 为什么行走距离这么短？

#### 1. 关节范围严重受限 ❌

| 关节 | 当前范围 | 标准范围 | 占比 |
|------|---------|---------|------|
| **Hip** | ±5° | ±60° | 8.3% |
| **Thigh (前)** | 30° | 290° | 10.3% |
| **Thigh (后)** | 30° | 290° | 10.3% |
| **Calf** | 40° | 108° | 37% |

**影响：**
- Hip ±5° → 腿部几乎无法横向移动
- Thigh 30° → 步幅极度受限
- Calf 40° → 膝关节活动受限

**结果：**
- 步幅很小
- 可能存在打滑
- 可能原地踏步

#### 2. 可能的其他原因

- **足部打滑**: 摩擦系数不够
- **步态不优**: MPC 参数未优化
- **控制器问题**: 力矩输出不够
- **仿真问题**: 时间步长或求解器设置

---

## 2. 鲁棒性测试 - 推力施加方式

### 测试设计

**目标**: 测试机器人从外部扰动中恢复的能力

**推力参数**:
- **大小**: 40 N (可调整 30-50 N)
- **方向**: 侧向 (Y 轴)
- **持续时间**: 0.1 秒
- **施加时间**: 5.0 秒 (行走过程中)

---

### 实现方式

#### 1. 施加推力

```python
# 在 t=5.0s 时施加推力
if step == push_step:
    # 使用 MuJoCo 的 xfrc_applied 施加外力
    data.xfrc_applied[trunk_body_id, 1] = 40.0  # Y 方向 40N
    logger.mark_push_applied(current_time)
    print(f"⚡ 在 {current_time:.2f}s 施加 40N 侧向推力")

# 0.1 秒后移除推力
elif step == push_step + int(0.1 / dt):
    data.xfrc_applied[trunk_body_id, :] = 0.0
```

**说明**:
- `xfrc_applied` 是 MuJoCo 的外力数组
- `trunk_body_id` 是机器人身体的 ID
- `[..., 1]` 表示 Y 轴方向（侧向）

#### 2. 检测恢复

```python
def check_recovery(self, current_time, height_threshold=0.15):
    """
    检查是否从推力中恢复
    
    判断标准:
    1. 身体高度 > 0.15 m (未摔倒)
    2. 高度稳定 (标准差 < 0.01 m)
    """
    # 检查推力后的高度
    heights_after_push = self.heights[push_step:]
    
    # 检查是否摔倒
    min_height = np.min(heights_after_push)
    if min_height < 0.15:  # 低于 15cm 认为摔倒
        return False
    
    # 检查是否恢复稳定
    if len(heights_after_push) > 50:  # 至少 0.5 秒后
        recent_heights = heights_after_push[-50:]
        height_std = np.std(recent_heights)
        if height_std < 0.01:  # 高度稳定
            return True
    
    return None  # 还在恢复中
```

## 3. 总结

### CoT 偏高的原因

1. **主要原因**: 行走距离太短 (只有期望的 45%)
   - 关节范围严重受限
   - 步幅很小

2. **次要原因**: 可能存在打滑或控制问题

### 改进建议

1. **放宽关节范围** (最重要)
   - Hip: ±5° → ±30°
   - Thigh: 30° → 200°
   - Calf: 40° → 108°

2. **优化 MPC 参数**

3. **增加摩擦系数**

### 鲁棒性测试

- ✅ 设计合理: 40N 侧向推力，0.1秒
- ✅ 评估全面: 成功率、恢复时间、高度下降
- ✅ 结果优秀: 100% 恢复成功率

**结论**: 虽然 CoT 偏高，但 MPC 控制器的鲁棒性表现优秀！
