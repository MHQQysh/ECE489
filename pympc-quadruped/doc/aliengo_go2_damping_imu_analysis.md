# Aliengo vs Go2 配置差异分析

## 1. 关节阻尼系数对比

### 1.1 实际数值

| 机器人 | 格式 | 阻尼系数 (damping) | 电枢惯性 (armature) | 摩擦损失 (frictionloss) |
|--------|------|-------------------|-------------------|----------------------|
| **Aliengo** | URDF | 0.2 | - | friction=0 |
| **Aliengo** | MJCF | 0.01 | 0.01 | - |
| **Go2** | URDF | 0.2 | - | friction=0 |
| **Go2** | MJCF (robot/go2) | 未定义（使用 MuJoCo 默认） | 未定义 | - |
| **Go2** | MJCF (scene_go2) | 0.1 | 0.01 | 0.2 |

### 1.2 修正后的表格

| 机器人 | URDF | MJCF (默认) | MJCF (scene) |
|--------|------|------------|-------------|
| **Aliengo** | damping=0.2 | damping=0.01 | - |
| **Go2** | damping=0.2 | 未定义 | damping=0.1 |

### 1.3 差异分析

**URDF 格式：**
- ✅ 两者完全相同：damping=0.2, friction=0
- 这是 URDF 标准格式，用于 ROS 等框架

**MJCF 格式：**
- ❌ **不完全相同**
- Aliengo MJCF: damping=0.01（较小，关节更自由）
- Go2 robot/go2/go2.xml: 未定义（使用 MuJoCo 默认值，通常为 0）
- Go2 scene_go2.xml: damping=0.1（中等，介于两者之间）

**为什么不同？**
1. **URDF → MJCF 转换差异**：从 URDF 转换到 MuJoCo 时，阻尼参数可能被调整
2. **仿真优化**：MuJoCo 中较小的阻尼可以提高仿真稳定性
3. **建模目的不同**：
   - Aliengo: 较早的模型，使用较小阻尼
   - Go2 scene: 更新的模型，使用更真实的阻尼值

## 2. IMU 传感器位置对比

### 2.1 实际位置

| 机器人 | Trunk 位置 (世界坐标) | IMU 位置 (相对 trunk) | IMU 绝对位置 | 质心位置 (COM) |
|--------|---------------------|---------------------|-------------|---------------|
| **Aliengo** | (0, 0, 0.6) | **(0, 0, 0)** | (0, 0, 0.6) | (0.00846, 0.00404, -0.00076) |
| **Go2** | (0, 0, 0.445) | **(-0.02557, 0, 0.04232)** | (-0.02557, 0, 0.48732) | (0.021112, 0, -0.005366) |

### 2.2 为什么 IMU 位置不同？

#### 原因 1: 真实硬件安装位置

**Go2 的 IMU 位置 (-0.02557, 0, 0.04232) 反映了真实硬件：**

```
X: -0.02557 m  → 向后偏移 2.5 cm（靠近机器人后部）
Y: 0 m         → 在中心线上（左右对称）
Z: 0.04232 m   → 向上偏移 4.2 cm（高于 trunk 原点）
```

**实际硬件原因：**
- IMU 芯片通常安装在电路板上
- 电路板位置受到机械结构限制
- 需要避开电机、电池等部件
- 可能为了更好的测量效果（远离振动源）

#### 原因 2: 建模精度差异

**Aliengo (0, 0, 0) - 简化建模：**
- ✅ 简化计算（IMU 在 trunk 局部坐标原点）
- ✅ 便于理解和调试
- ❌ 不完全符合真实硬件
- 适用场景：研究、教学、算法验证

**Go2 (-0.02557, 0, 0.04232) - 精确建模：**
- ✅ 更接近真实硬件
- ✅ 更准确的动力学仿真
- ✅ 仿真到实物迁移更容易
- 适用场景：产品开发、硬件测试、精确仿真

#### 原因 3: 对传感器数据的影响

**IMU 位置偏移的影响：**

1. **角速度测量（陀螺仪）：**
   - 不受位置影响（角速度在刚体上处处相同）
   - ✅ 两者测量结果相同

2. **线加速度测量（加速度计）：**
   - 受位置影响（不同位置的线加速度不同）
   - 公式：`a_imu = a_com + ω × (ω × r) + α × r`
   - 其中 r 是 IMU 到质心的距离
   - ⚠️ Go2 的 IMU 偏离中心，测量值会包含旋转引起的离心加速度

3. **姿态测量（四元数）：**
   - 不受位置影响（姿态是刚体属性）
   - ✅ 两者测量结果相同

### 2.3 数值对比

**IMU 到质心的距离：**

```python
# Aliengo
r_aliengo = sqrt((0 - 0.00846)² + (0 - 0.00404)² + (0 - (-0.00076))²)
r_aliengo ≈ 0.0095 m ≈ 0.95 cm

# Go2
r_go2 = sqrt((-0.02557 - 0.021112)² + (0 - 0)² + (0.04232 - (-0.005366))²)
r_go2 ≈ 0.0567 m ≈ 5.67 cm
```

**结论：**
- Aliengo IMU 距离质心约 1 cm（非常接近）
- Go2 IMU 距离质心约 5.7 cm（有明显偏移）
- Go2 的加速度计读数会受到更大的旋转影响

## 4. 实际影响分析

### 4.1 对控制算法的影响

**使用 Aliengo 模型：**
- ✅ 状态估计更简单（IMU 在中心）
- ✅ 算法开发更快
- ⚠️ 迁移到实物时需要调整

**使用 Go2 模型：**
- ⚠️ 需要考虑 IMU 位置偏移
- ✅ 仿真结果更接近实物
- ✅ 算法可以直接迁移

### 4.2 对传感器融合的影响

**加速度计数据处理：**

```python
# Aliengo: 简化处理
a_com = a_imu  # 直接使用 IMU 读数

# Go2: 需要补偿
r_imu = np.array([-0.02557, 0, 0.04232])  # IMU 相对 trunk 的位置
a_com = a_imu - np.cross(omega, np.cross(omega, r_imu)) - np.cross(alpha, r_imu)
```

### 4.3 对动力学仿真的影响

**旋转运动时：**
- Aliengo: IMU 加速度 ≈ 质心加速度
- Go2: IMU 加速度 = 质心加速度 + 旋转引起的加速度

**示例（机器人快速转向）：**
```
角速度 ω = 2 rad/s
IMU 偏移 r = 0.057 m
离心加速度 = ω² × r = 4 × 0.057 = 0.228 m/s²
```

Go2 的 IMU 会额外测量到 0.228 m/s² 的加速度。

### 5.3 如何统一 IMU 位置？

**方案 1: 修改 Go2 模型，将 IMU 移到中心**
```xml
<!-- robot/go2/go2.xml -->
<site name="imu" pos="0 0 0" group="5"/>  <!-- 改为 (0, 0, 0) -->
```

**方案 2: 在代码中补偿 IMU 位置**
```python
def compensate_imu_position(a_imu, omega, alpha, r_imu):
    """
    将 IMU 位置的加速度转换为质心加速度
    """
    a_com = a_imu - np.cross(omega, np.cross(omega, r_imu)) - np.cross(alpha, r_imu)
    return a_com
```

## 7. 常见问题

### Q1: 为什么 URDF 和 MJCF 的阻尼不同？
**A:** 
- URDF 的 damping=0.2 是关节阻尼（joint damping）
- MJCF 的 damping=0.01 是默认值，可能还有其他阻尼来源（如 armature）
- 两者的物理意义和单位可能不完全相同

### Q2: IMU 位置偏移会影响控制吗？
**A:** 
- 对姿态控制：影响很小（角速度和姿态不受位置影响）
- 对位置控制：有影响（加速度测量包含旋转分量）
- 建议：在状态估计中补偿 IMU 位置

### Q3: 应该使用哪个模型？
**A:**
- 学习/研究：Aliengo（简单）
- 产品开发：Go2（精确）
- 算法验证：两者都可以

### Q4: 如何验证 IMU 位置是否正确？
