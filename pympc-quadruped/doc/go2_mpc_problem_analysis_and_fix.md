## 问题根源

### 发现的问题

通过对比 `config/robot_configs.py` 中的配置，发现 **Go2 的摆动足控制增益严重偏低**。

### 配置对比

| 参数 | Aliengo | Go2 (修改前) | 差异 |
|------|---------|-------------|------|
| **Kp_swing (X)** | 200.0 | 20.0 | Go2 仅为 Aliengo 的 10% |
| **Kp_swing (Y)** | 200.0 | 20.0 | Go2 仅为 Aliengo 的 10% |
| **Kp_swing (Z)** | 200.0 | 40.0 | Go2 仅为 Aliengo 的 20% |
| **Kd_swing (X)** | 20.0 | 1.0 | Go2 仅为 Aliengo 的 5% |
| **Kd_swing (Y)** | 20.0 | 1.0 | Go2 仅为 Aliengo 的 5% |
| **Kd_swing (Z)** | 20.0 | 2.0 | Go2 仅为 Aliengo 的 10% |
| **swing_height** | 0.1 m | 0.06 m | Go2 低 40% |

---

## 为什么增益太低会导致问题？

### 1. Kp (比例增益) 太低

**Kp 的作用：** 控制位置误差的响应强度

**Kp 太低的后果：**
- ❌ 摆动足无法快速跟踪目标位置
- ❌ 位置误差大
- ❌ 足部落地位置不准确
- ❌ 步态不稳定

**示例：**
```
目标位置: (0.2, 0.1, 0.3)
实际位置: (0.15, 0.08, 0.25)
误差:     (0.05, 0.02, 0.05)

Kp = 200: 控制力 = 200 × 0.05 = 10 N (足够)
Kp = 20:  控制力 = 20 × 0.05 = 1 N (太小！)
```

### 2. Kd (微分增益) 太低

**Kd 的作用：** 提供阻尼，抑制振荡

**Kd 太低的后果：**
- ❌ 摆动足振荡
- ❌ 超调严重
- ❌ 不稳定
- ❌ 能量浪费

**示例：**
```
足部快速接近目标时：

Kd = 20: 阻尼力 = 20 × 速度 (足够阻尼，平滑停止)
Kd = 1:  阻尼力 = 1 × 速度 (阻尼不足，振荡)
```

### 3. swing_height 太低

**swing_height 的作用：** 摆动足抬起的高度

**太低的后果：**
- ❌ 容易碰到地面
- ❌ 步态不自然
- ❌ 对地面不平整敏感

## 修复方案

### 修改内容

**文件：** `config/robot_configs.py`

**修改前：**
```python
class Go2Config(RobotConfig):
    # ...
    swing_height = 0.06
    Kp_swing = np.diag([20.0, 20.0, 40.0])
    Kd_swing = np.diag([1.0, 1.0, 2.0])
```

**修改后：**
```python
class Go2Config(RobotConfig):
    # ...
    swing_height = 0.08
    Kp_swing = np.diag([180.0, 180.0, 180.0])
    Kd_swing = np.diag([18.0, 18.0, 18.0])
```

### 修改理由

#### 1. Kp_swing: 20 → 180

**原因：**
- Go2 虽然比 Aliengo 轻 23.5%，但不应该降低 90% 的增益
- 摆动足控制需要足够的刚度来准确跟踪轨迹
- 180 接近 Aliengo 的 200，考虑到 Go2 更轻，略微降低是合理的

**计算依据：**
```
Aliengo: 质量 9.042 kg, Kp = 200
Go2:     质量 6.921 kg, Kp = 180

比例: 180/200 = 0.9
质量比: 6.921/9.042 = 0.765

Kp 降低 10%，质量降低 23.5%
→ 单位质量的刚度实际增加了约 18%
→ 这是合理的，因为 Go2 腿更短，需要更快的响应
```

#### 2. Kd_swing: 1 → 18

**原因：**
- 保持 Kd/Kp = 0.1 的比例（临界阻尼附近）
- Kd = 18 提供足够的阻尼，防止振荡
- 与 Aliengo 的 Kd/Kp = 20/200 = 0.1 一致

**阻尼比分析：**
```
临界阻尼: Kd_critical = 2 × sqrt(Kp × m)

对于 Go2 (假设足部质量 ~0.2 kg):
Kd_critical ≈ 2 × sqrt(180 × 0.2) ≈ 12

使用 Kd = 18 > 12 → 过阻尼（稳定，无振荡）
使用 Kd = 1 << 12 → 欠阻尼（振荡）
```

#### 3. swing_height: 0.06 → 0.08

**原因：**
- 0.06 m 太低，容易碰到地面
- 0.08 m 是 Aliengo 0.1 m 的 80%，考虑到 Go2 腿更短是合理的
- 提供足够的离地间隙

**对比：**
```
Aliengo: 腿长 ~0.5 m, swing_height = 0.1 m (20%)
Go2:     腿长 ~0.4 m, swing_height = 0.08 m (20%)
→ 保持相同的比例
```

## 总结

### 问题根源
Go2 的摆动足控制增益（Kp_swing, Kd_swing）设置过低，导致：
- 位置跟踪误差大
- 摆动足振荡
- 步态不稳定

### 解决方案
将增益提高到合理水平：
- Kp_swing: 20 → 180
- Kd_swing: 1 → 18
- swing_height: 0.06 → 0.08

### 预期结果
- ✅ 摆动足准确跟踪轨迹
- ✅ 无振荡
- ✅ 步态稳定
